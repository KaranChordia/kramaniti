# Quality Gates

Before an agent considers a task "Done" and hands it off or presents it for human review, the work must pass these quality gates.

## 1. The Strategy Gate
- [ ] Does this align with "Strategy before tools, systems before scale, content after clarity"?
- [ ] Is it consistent with the latest files in `03_brand_strategy/`?

## 2. The Voice Gate
- [ ] Is the language clear, practical, and grounded?
- [ ] Are generic AI buzzwords (e.g., "revolutionary", "unlock the future", "automate everything") removed?
- [ ] Does it sound like an experienced systems partner rather than a generic tech agency?

## 3. The Proof Gate (Mandatory for public content)
- [ ] Are all claims (metrics, clients, testimonials) verified in `02_founder_context/` or approved?
- [ ] If unverified, are they properly labeled `[Unverified]` or softened to category-level language?
- [ ] Did the Proof and Governance Auditor review public-facing claims?

## 4. The System Gate
- [ ] Was the relevant `AGENTS.md` read before executing?
- [ ] Were the correct files updated?
- [ ] Is the next step clear for the human or the next agent?
- [ ] If a strategic decision was made, is it logged in `09_reviews/decision-log/decisions.md`?
